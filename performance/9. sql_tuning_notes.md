# SQL Tuning Notes

## Key Practices

* Capture SQL_ID and PLAN_HASH_VALUE
* Compare historical and current plans
* Validate whether a SQL Profile or Baseline is influencing execution
* Use ASH to identify concurrency and wait events
* Confirm improvements with repeated execution and timing

## Typical Use Cases

* Sudden regression after patching
* Plan instability
* Long-running SQL
* Baseline/profile validation
