# RAC Service Failover Test

## Objective

Validate service-level high availability in RAC.

## Steps

### 1. Identify Service

```bash
srvctl status service -db <db>
```

### 2. Stop Instance Hosting Service

```bash
srvctl stop instance -d <db> -i <instance>
```

### 3. Validate Service Relocation

```bash
srvctl status service -db <db>
```

### 4. Validate Application Connectivity

* Ensure reconnection occurs automatically

## Expected Result

Service should relocate to available instance without downtime.

## Notes

* Configure services with preferred/available instances
* Use server pools if needed
